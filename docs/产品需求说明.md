# 产品需求说明：产品加工流程管理系统

## 1. 项目背景

飞航厂房产品加工过程涉及打磨、装配、喷漆、包覆等工序。产品在工序间流转时，如果依赖人工登记或口头同步，容易出现当前状态不透明、滞留超时发现不及时、管理层难以及时掌握进度等问题。

本系统用于记录和查看产品加工流转状态。现场每个工序只配置固定扫码枪，不配置电脑。产品到达某个工序后，扫码枪扫描产品上已有二维码，后台根据扫码枪来源识别工序，根据扫码内容识别产品，并记录产品进入该工序的时间。

二维码由其他系统或其他环节单独打印并贴到产品上，本系统不负责二维码生成和打印，只负责扫码识别、流转记录、状态看板和超时预警。

## 2. 建设目标

- 后台系统只面向管理员使用。
- 管理员通过账号密码登录后台。
- 管理员维护产品、工序、扫码枪和预警。
- 产品可手动单条录入，暂不做批量导入。
- 二维码由外部方式打印并贴到产品上，本系统不提供二维码打印功能。
- 扫码枪扫码后，系统识别产品并记录进入当前工序。
- 每次扫码即表示产品进入该扫码枪绑定的工序。
- 看板实时显示产品当前加工状态。
- 大屏看板页面无需登录，电视或任意操作系统浏览器输入访问地址即可展示。
- 单工序滞留时间超过 3 天自动预警。

## 3. 用户与权限

当前阶段系统只有一个使用角色：管理员。

| 角色 | 权限范围 |
| --- | --- |
| 管理员 | 登录后台，维护管理员账号密码，维护产品、工序、扫码枪，查看看板和预警 |

当前阶段不做工序负责人登录，不做多角色权限划分。工序负责人可以作为产品或工序上的业务字段展示，但不是系统登录用户角色。

大屏看板 `/screen` 是公开只读页面，不要求登录，不提供任何编辑操作。

## 5. 核心流程

### 5.1 产品建档

1. 管理员登录后台。
2. 进入产品管理。
3. 手动录入单个产品信息。
4. 系统保存产品档案，初始状态为待加工或未开始。

### 5.2 二维码来源

1. 产品二维码由其他地方单独打印并贴到产品上。
2. 本系统不生成二维码、不打印二维码、不维护二维码模板。
3. 本系统只约定扫码内容优先使用产品型号。
4. 若后续存在同一产品型号多件产品同时流转，可扩展为“产品型号 + 流水号”或系统产品编号。

### 5.3 工序与扫码枪绑定

默认工序：

1. 打磨
2. 装配
3. 喷漆
4. 包覆
5. 完工

每个工序配置固定扫码枪。扫码枪需要在后台登记：

- 扫码枪编号
- 扫码枪名称
- 绑定工序
- 扫码枪 IP 或网络标识
- 安装位置
- 是否启用

当前默认扫码枪：

| 扫码枪编号 | 扫码枪名称 | 绑定工序 | 安装位置 |
| --- | --- | --- | --- |
| `SCAN-DM-01` | 打磨扫码枪 | 打磨 | 打磨工序 |
| `SCAN-ZP-01` | 装配扫码枪 | 装配 | 装配工序 |
| `SCAN-PQ-01` | 喷漆扫码枪 | 喷漆 | 喷漆工序 |
| `SCAN-BF-01` | 包覆扫码枪 | 包覆 | 包覆工序 |

### 5.4 扫码流转

1. 产品到达某个工序。
2. 现场人员使用该工序固定扫码枪扫描产品二维码。
3. 扫码枪把扫码内容和扫码枪标识上传到后台。
4. 后台根据扫码枪标识判断当前工序。
5. 后台根据扫码内容匹配产品。
6. 系统记录产品进入该工序的时间。
7. 产品当前状态更新为该工序进行中。
8. 上一工序滞留统计结束，当前工序滞留计时开始。

### 5.5 手动调整工序

当产品真实生产流程已经进入某个工序，但该工序扫码枪异常、漏扫或误扫时，管理员可以在后台产品详情中手动调整工序。

示例：产品打磨完成后已经进入装配，但装配扫码枪故障，管理员可以将该产品手动调整到装配工序。系统需要更新产品当前工序和当前工序进入时间，并生成一条来源为“管理员调整”的流转记录。

手动调整工序不是删除或覆盖历史，而是补充一条可追溯的流转记录。

### 5.6 超时预警

- 每个产品在单个工序滞留不能超过 3 天。
- 从产品扫码进入该工序的时间开始计时。
- 超过 72 小时仍未扫码进入下一工序，则标记为超时。

预警展示位置：

- 后台首页
- 预警中心
- 大屏看板

## 6. 功能模块

### 6.1 管理员登录

- 管理员账号密码登录。
- 管理员修改密码。
- 管理员账号启用、停用。
- 当前阶段只做管理员，不做工序负责人账号。

### 6.2 产品管理

- 新增产品。
- 编辑产品。
- 删除产品或停用产品。
- 查看产品详情。
- 查看产品流转记录。流转记录以列表展示该产品已经经过的全部工序。
- 产品详情支持“手动调整工序”，用于扫码枪异常、漏扫或误扫后的管理员进度修正。
- 按产品名称、产品型号、流水号、当前工序、状态查询。
- 暂不做批量导入。
- 暂不做二维码生成和打印。

### 6.3 工序管理

- 工序维护。
- 工序顺序配置。
- 每个工序标准时限配置，默认 72 小时。
- 启用或停用工序。

### 6.4 扫码枪管理

- 扫码枪建档。
- 绑定工序。
- 维护扫码枪 IP、编号和安装位置。
- 启用或停用扫码枪。
- 查看扫码枪最近上报时间和状态。

### 6.5 扫码流转

- 接收扫码枪上报数据。
- 识别扫码内容。
- 根据扫码枪判断工序。
- 根据扫码内容匹配产品。
- 自动生成“扫码进入”流转记录。
- 管理员手动调整工序时，自动生成“管理员调整”流转记录。
- 流转记录字段包括经过工序、进入时间、来源、扫码枪或操作来源、扫码内容或说明。
- 重复扫码校验。
- 越序扫码提示或拦截。
- 操作日志追溯。

### 6.6 实时数据看板

- 产品总数。
- 加工中数量。
- 已完工数量。
- 超时数量。
- 各工序产品数量。
- 每个产品当前工序、进入时间、滞留时长。
- 超时产品高亮显示。
- 大屏看板独立访问，不要求登录，适合车间电视全屏展示。
- 当前大屏看板顶部四个指标以屏幕整体居中展示。
- 当前大屏看板主体展示打磨、装配、喷漆、包覆四个工序。
- 每个工序顶部展示工序名称、在制数量和超时数量。
- 每个工序内部使用两列表格展示产品型号和当前工序已用时。
- 正常产品使用白色文字，超时产品使用亮橙色文字。
- 工序内部列表隐藏滚动条，但保留内部滚动能力。

### 6.7 预警中心

- 超时预警列表。
- 按工序、产品、时间范围筛选。
- 预警处理状态。
- 处理说明。
- 预警历史查询。

## 7. 当前阶段不做

- 不做工序负责人登录。
- 不做多角色权限体系。
- 不做任务单管理。
- 不做独立资产台账模块。
- 不做批量导入。
- 不做二维码生成。
- 不做二维码打印。
- 不做二维码标签模板。
- 暂不做报表分析。
