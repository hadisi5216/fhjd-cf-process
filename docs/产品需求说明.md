# 产品需求说明

## 1. 项目概述

### 1.1 项目名称

产品加工流程管理系统

### 1.2 使用单位

北京飞航吉达航空科技有限公司

### 1.3 建设背景

厂房产品需要经过打磨、装配、喷漆、包覆等加工环节。现场每个工序只安装固定扫码枪，不配置电脑。产品到达工序后扫描已有二维码，系统据此记录当前工序、进入时间和滞留时长，供管理员和车间大屏实时查看。

二维码由其他系统或其他环节生成、打印并贴到产品上。本系统不负责二维码生成、标签设计和打印。

## 2. 建设目标

- 统一维护产品、工序和扫码枪资料。
- 通过扫码自动记录产品当前工序和完整历史。
- 及时展示每道工序正在加工的产品及已用时。
- 产品在当前工序超时后自动预警。
- 扫码枪故障、漏扫或误扫时允许管理员人工修正，并保留审计记录。
- 车间电视使用任意现代浏览器直接打开公开大屏，无需登录。

## 3. 用户与权限

当前只有“系统管理员”一种角色。

管理员可以登录后台，维护密码、产品、工序、扫码枪和系统设置，查看产品详情、流转记录、首页与预警中心，并手动调整产品工序。

公开大屏 `/screen` 无需登录，只提供只读展示，不提供任何维护操作。当前不建设工序负责人账号和多角色权限体系。

## 4. 产品信息

产品由管理员手动单条录入，录入和编辑时不要求输入序号。系统当前不支持 Excel、任务单或其他批量导入。

| 字段 | 必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| 产品名称 | 是 | 产品或部件名称 | 12号舵舱体 |
| 产品型号 | 是 | 扫码识别内容 | J/CLL9-12A-101S1.1 |
| 流水号 | 否 | 批次或生产流水号 | 2606003 |
| 数量 | 是 | 默认 1 | 1 |
| 单位 | 是 | 默认“件” | 件 |
| 备注 | 否 | 补充说明 | 返工件 |
| 产品工艺 | 否 | 自由编辑的制作参考，多行文本，最多 5000 字 | 下料→打磨→检验 |

当前使用产品型号匹配产品。同一型号存在多件需要分别追踪时，应由外部二维码使用唯一型号后缀区分，例如 `J/CLL9-12A-101S1.1+01`、`J/CLL9-12A-101S1.1+02`。

每个产品可以上传多份产品图纸。图纸列表保存原始文件名、格式、大小和上传时间；PDF 图纸支持在浏览器新标签中在线预览，其他格式支持下载。删除产品图纸需要管理员二次确认。

产品工艺是产品档案中的独立说明，仅供制作产品时查看。它不关联工序配置、扫码枪、当前工序、流转记录、超时预警或大屏统计，编辑产品工艺不会改变产品实际流转状态。

## 5. 工序与扫码枪

默认工序和扫码枪如下。IP 是后台管理和无 URL 标识上报时的兜底信息，正式上报优先使用扫码枪编号。

| 编号 | 扫码枪 | 绑定工序 | 默认 IP | HTTP 路径 |
| --- | --- | --- | --- | --- |
| `DM` | 打磨扫码枪 | 打磨 | `192.168.1.11` | `/api/scanners/scan?name=DM` |
| `ZP` | 装配扫码枪 | 装配 | `192.168.1.12` | `/api/scanners/scan?name=ZP` |
| `PQ` | 喷漆扫码枪 | 喷漆 | `192.168.1.13` | `/api/scanners/scan?name=PQ` |
| `BF` | 包覆扫码枪 | 包覆 | `192.168.1.14` | `/api/scanners/scan?name=BF` |
| `WG` | 完工扫码枪 | 完工 | `192.168.1.15` | `/api/scanners/scan?name=WG` |

扫码枪管理字段包括编号、名称、绑定工序、IP、安装位置、是否启用和最近上报时间。HTTP 为短连接，系统不判断在线或离线状态。

## 6. 业务流程

### 6.1 产品建档

1. 管理员登录后台并新增产品。
2. 产品初始状态为“待加工”，没有当前工序和进入时间。
3. 外部系统生成并打印含产品型号的二维码，贴到对应产品上。

### 6.2 扫码流转

1. 产品到达某工序，现场人员使用该工序固定扫码枪扫码。
2. 扫码枪通过 HTTP POST 上报产品型号，并在 URL 中携带自身编号。
3. 系统根据扫码枪编号确定工序，根据请求体中的产品型号匹配产品。
4. 系统写入流转记录，并根据规则更新产品当前工序、状态和进入时间。
5. 非重复扫码完成后，系统主动通知公开大屏刷新；5 秒自动刷新继续作为兜底。

### 6.3 工序规则

- 打磨、装配、喷漆、包覆之间没有固定先后顺序，不做越序校验。
- 产品扫描哪道工序的扫码枪，就进入哪道工序，并从本次扫码时间开始计时。
- 扫描工序与产品当前工序相同时，本次记录标记为“重复扫码”。
- 重复扫码保留历史记录，但不改变当前工序、产品状态和当前工序进入时间。
- 产品进入其他工序后再次返回原工序，属于重新进入，不是重复扫码，并从本次扫码时间重新计时。
- 已完工产品扫描任意加工工序后可以重新进入加工，状态恢复为“加工中”。
- 完工扫码枪扫描后，产品当前工序变为“完工”，状态变为“已完工”，并立即从大屏四个在制工序中移除。
- 同一产品并发扫码时，系统必须串行处理，避免工序状态和重复标记相互覆盖。

### 6.4 手动调整工序

扫码枪异常、漏扫或误扫，但产品实际已经进入某工序时，管理员可在产品详情中执行“手动调整工序”。

手动调整会更新产品当前工序、状态和进入时间，并新增来源为“管理员调整”的历史记录，不删除或覆盖既有记录。手动调整到完工时产品状态为“已完工”，调整到其他工序时状态为“加工中”。

### 6.5 流转记录

产品详情中的流转记录是产品经过工序的完整历史，不是当前工序的重复说明。记录按时间正序展示，至少包含：

- 经过工序。
- 记录时间。
- 来源：扫码进入或管理员调整。
- 是否重复扫码。
- 扫码枪或操作来源。
- 扫码内容或调整说明。

每次扫码都必须留存记录，包括重复扫码。重复扫码在页面中显示明确标记。

### 6.6 超时预警

- 打磨、装配、喷漆、包覆默认时限为 72 小时，工序管理中可单独调整。
- 从产品首次进入当前工序的时间开始计时，重复扫码不重置计时。
- 达到工序配置时限且仍在该工序时生成超时预警。
- 预警显示在后台首页、预警中心和公开大屏。
- 完工工序不计算滞留超时。

## 7. 功能模块

### 7.1 登录与系统设置

- 管理员账号密码登录。
- JWT 登录有效期 8 小时，失效后返回登录页。
- 修改管理员密码。
- 开关大屏模拟数据。

### 7.2 后台首页

- 展示产品总数、加工中、已完工和超时数量。
- 统计卡可跳转至带相应筛选条件的产品管理。
- 展示正在加工的产品列表，整行可进入产品详情。
- 超时提示不写死“72 小时”，以工序配置时限为准。

### 7.3 产品管理

- 手动新增、编辑、删除、查询产品。
- 按关键词、当前工序和状态筛选。
- 将当前筛选条件匹配的全部产品导出为 Excel，不受列表 200 条展示上限影响。
- 点击整行、产品名称或“详情”按钮进入产品详情。
- 查看基础信息和完整流转记录。
- 查看、编辑或清空产品工艺说明，保留录入的多行格式。
- 上传、查看、下载和删除产品图纸；PDF 支持新标签在线预览。
- 管理员手动调整当前工序。

### 7.4 工序管理

- 新增、编辑、删除和启停工序。
- 设置展示顺序和超时时限。
- 展示顺序只影响界面排序，不限制真实流转顺序。

### 7.5 扫码枪管理

- 新增、编辑、删除和启停扫码枪。
- 绑定工序并维护编号、IP 和安装位置。
- 显示最近上报时间，不显示在线状态。

### 7.6 预警中心

- 展示超时预警及对应产品、工序、进入时间和处理状态。
- 支持填写处理说明并标记已处理。
- 列表按已用时从长到短展示。

### 7.7 公开大屏

- 无需登录，地址为 `/screen`。
- 顶部展示标题、公司名称、四项统计和当前时间。
- 主体按打磨、装配、包覆、涂装的顺序展示四个在制工序，不展示完工产品；看板“涂装”对应后台“喷漆”工序。
- 每道工序显示在制数量、超时数量，以及产品型号和当前工序已用时。
- 产品按已用时从长到短排列，超时使用亮橙色高亮。
- 数据不足一屏时保留空表格行；超过一屏时工序内部可滚动且不显示滚动条。
- 后台可切换模拟数据。模拟模式每个工序 40 条，共 160 条，仅用于展示测试。
- 非重复扫码后通过 SSE 立即刷新，同时保留 5 秒自动刷新。

## 8. 当前不做

- 工序负责人账号和多角色权限。
- 任务单管理和独立设备台账。
- Excel 或其他批量导入。
- 二维码生成、打印和标签模板。
- 报表分析。
- 企业微信、钉钉等外部协同集成。

## 9. 验收要点

1. 管理员可以登录、修改密码并完成所有后台维护。
2. 产品可手动新增，并可按工序、状态和关键词筛选。
3. 五台默认扫码枪可通过各自 URL 上报产品型号并正确识别工序。
4. 任意工序之间可直接流转，不受排序限制。
5. 连续扫描当前工序只新增重复记录，不重置计时；切换后返回原工序正常重新计时。
6. 完工扫码后产品状态变为已完工，并立即从大屏移除。
7. 管理员手动调整后，当前工序和流转历史同时正确更新。
8. 产品超过工序时限后生成预警，后台和大屏显示一致。
9. 产品详情可上传多份图纸，PDF 可在新标签预览，其他图纸可下载，删除后文件和记录同时移除。
10. 产品详情可编辑和清空产品工艺，保存后不改变当前工序且不新增流转记录。
11. 产品列表导出结果与当前关键词、工序和状态筛选一致，未设置筛选时导出全部产品。
12. `/screen` 未登录可访问，扫码后即时刷新，SSE 异常时仍能在 5 秒内更新。
