# 实施方案：产品加工流程管理系统

## 1. 总体方案

系统采用“产品二维码识别 + 工位固定扫码枪 + 后台管理 + 实时看板”的方式建设。

管理员在后台维护产品信息、工序信息和扫码枪信息。打磨、装配、喷漆、包覆等工序各自配置固定扫码枪，现场不配置电脑。产品流转到某个工序时，现场人员直接使用该工序扫码枪扫描产品上已有二维码。后台根据扫码枪编号或 IP 判断产品进入的工序，并自动记录进入时间。

二维码由其他地方单独打印并贴到产品上，本系统不负责二维码生成和打印，只负责扫码识别和流转记录。

每个产品在当前工序的滞留时间从扫码进入该工序开始计算。超过 3 天仍未扫码进入下一工序时，系统在后台管理页面和大屏看板中预警。

## 2. 当前实施边界

### 2.1 当前阶段包含

- 管理员账号密码登录。
- 管理员密码维护。
- 产品手动单条录入。
- 产品编辑、删除、查询。
- 工序维护。
- 扫码枪维护。
- 扫码枪扫码上报。
- 产品流转记录。
- 后台首页。
- 大屏看板。
- 预警中心。
- Docker 部署。

### 2.2 当前阶段不包含

- 不做工序负责人登录。
- 不做多角色权限体系。
- 不做任务单管理。
- 不做独立资产台账模块。
- 不做批量导入。
- 不做二维码生成。
- 不做二维码打印。
- 不做二维码标签模板。
- 暂不做报表分析。

## 3. 硬件建议

### 3.1 固定式扫码枪

本项目每个工序只配置扫码枪，不配置电脑。因此扫码枪需要能直接联网并把扫码结果上传到后台。

建议选择：

- 支持二维码识读。
- 支持网口通信。
- 支持标准 PoE 供电，建议确认 IEEE 802.3af 或 802.3at。
- 支持配置扫码枪 IP、服务器 IP、端口和上报格式。
- 支持 HTTP、TCP Socket 或其他明确可对接协议。
- 支持弱光、污损、反光场景扫码。
- 车间环境建议具备一定防尘、防水能力。

不建议在本阶段采用普通 USB 扫码枪，因为工序现场没有电脑，USB 扫码枪无法独立上报数据。

### 3.2 PoE 网络

建议每个扫码点位布置一根网线，通过 PoE 交换机同时完成供电和数据传输。

网络要求：

- 每个扫码枪有固定 IP 或可识别的硬件编号。
- 后台能识别扫码枪来源。
- 扫码枪到后台服务网络连通稳定。
- PoE 交换机端口数量预留扩展余量。

### 3.3 大屏看板

用途：

- 展示车间产品加工状态。
- 展示各工序产品数量。
- 展示超时预警。
- 展示产品当前工序和滞留时长。

建议：

- 使用车间电视或 LED 大屏。
- 浏览器全屏访问看板页面。
- 大屏看板不做登录验证，任意操作系统的电视、工控机或普通电脑浏览器输入访问地址即可展示。
- 大屏页面只读展示，不提供产品维护、工序配置、扫码枪管理等操作。
- 支持自动刷新和异常高亮。
- 当前看板顶部左侧显示“产品加工流程实时看板”和公司名称，中间显示产品总数、加工中、已完工、已超时四个统计卡，右侧显示当前时间和刷新倒计时。
- 当前看板主体按打磨、装配、喷漆、包覆四个工序分栏展示，每个工序显示在制数量、超时数量，以及产品型号和已用时表格。
- 当前看板超时统一使用亮橙色提示，正常产品使用白色文字。

## 5. 关键规则

### 5.1 二维码规则

二维码由其他地方单独打印，本系统不负责打印。

本系统只约定扫码枪上传的扫码内容用于匹配产品。当前优先使用产品型号。若后续出现同一产品型号对应多件产品同时流转，可扩展为“产品型号 + 流水号”或系统内部产品编号。

### 5.2 工序识别

工序不由现场人员选择，而由扫码枪绑定关系确定。

示例：

- 打磨扫码枪扫到产品二维码，系统记录产品进入打磨。
- 装配扫码枪扫到产品二维码，系统记录产品进入装配。
- 喷漆扫码枪扫到产品二维码，系统记录产品进入喷漆。
- 包覆扫码枪扫到产品二维码，系统记录产品进入包覆。

### 5.3 工序超时

默认规则：

- 标准滞留上限：3 天。
- 换算方式：72 小时。
- 触发条件：当前工序进入时间距当前时间超过 72 小时，且产品尚未扫码进入下一工序。

可扩展为每个工序单独配置标准时限。

### 5.4 流转校验

建议校验：

- 产品按配置好的工序顺序扫码流转。
- 同一产品在同一工序短时间重复扫码时，系统提示或自动忽略重复记录。
- 如果产品在后续工序扫码，上一工序自动结束滞留统计。
- 已完工产品不能重复流转，除非管理员重新打开或发起返工。
- 每次扫码都记录扫码枪、绑定工序、接收时间和产品信息。

### 5.5 手动调整工序

如果产品真实生产流程已经进入某工序，但现场扫码枪故障、漏扫或误扫，管理员可在产品详情中使用“手动调整工序”修正进度。

示例：产品打磨完成后实际进入装配，但装配扫码枪故障，管理员可将产品手动调整为装配工序。系统需要同步更新产品当前工序、当前工序进入时间，并新增一条来源为“管理员调整”的流转记录。

手动调整记录与扫码进入记录展示在同一流转记录列表中，用来源字段区分。

## 6. 分阶段实施

### 第一阶段：基础可用版

目标：

- 完成管理员登录。
- 完成产品管理，支持单条手动录入。
- 完成工序和扫码枪绑定关系。
- 支持固定扫码枪上报扫码数据。
- 支持扫码进入工序。
- 支持管理员手动调整工序，并生成管理员调整流转记录。
- 产品详情展示全部历史流转记录列表。
- 提供基础实时看板。
- 支持 3 天超时预警。

建议范围：

- 工序固定为打磨、装配、喷漆、包覆、完工。
- 每个工序先配置 1 台网口 PoE 扫码枪。
- 预警在后台首页、预警中心和大屏看板展示。

### 第二阶段：增强版

目标：

- 支持多个扫码枪绑定同一工序。
- 支持扫码枪在线状态监控。
- 增加预警处理闭环。
- 增加报表分析。
- 增加产品流转历史详情。

### 第三阶段：优化版

目标：

- 支持返工或重新流转。
- 支持更多工序扩展。
- 支持与 MES、ERP 或仓储系统对接。
- 支持产品加工周期统计和瓶颈分析。
- 视需要增加批量导入或二维码打印，但不属于当前阶段。

## 7. 验收指标

- 管理员能登录后台并修改密码。
- 管理员能手动录入产品。
- 管理员能维护产品、工序和扫码枪。
- 固定扫码枪能上传扫码内容。
- 后台能识别扫码枪来源并匹配对应工序。
- 后台能根据扫码内容识别产品。
- 扫码后能自动生成产品进入工序的流转记录。
- 管理员手动调整工序后能生成来源为“管理员调整”的流转记录。
- 产品详情能查看该产品已经经过的全部工序历史。
- 看板能展示产品当前工序、进入时间和滞留时长。
- 超过 3 天的产品能自动进入预警列表。
- 大屏看板不登录即可访问，且只读展示。
