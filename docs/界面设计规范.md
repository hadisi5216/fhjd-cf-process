# 界面设计规范

## 1. 设计目标

系统界面分为两种使用场景：

- 管理后台：面向管理员高频维护，强调干净、简洁、稳定和信息效率。
- 公开大屏：面向车间电视远距离查看，强调高对比、实时状态和同屏数据量。

系统名称统一为“产品加工流程管理系统”，业务对象统一称“产品”。不得再次出现“设备加工流程”“设备管理”“任务单”或 `Factory Process Management` 等旧文案。

## 2. 全局规范

### 2.1 字体与层级

- 使用系统中文无衬线字体，避免额外字体加载失败。
- 页面标题、卡片标题、正文、辅助信息形成明确层级，不使用过多字号。
- 表格操作文字保持常规字号与中等字重，不使用醒目的粗体链接。
- 数字使用等宽数字特性，避免时间和统计数字变化导致布局跳动。
- 字间距统一为 0，不使用负字间距。

### 2.2 颜色

后台主色为 `#126e78`，链接、按钮、输入焦点和选中状态均从 Ant Design 全局主题统一配置。禁止在单个页面重新使用默认蓝色焦点色。

| 用途 | 颜色 |
| --- | --- |
| 后台主色 | `#126e78` |
| 主色悬停 | `#0f5f67` |
| 后台页面背景 | 浅灰 |
| 后台内容背景 | 白色 |
| 已完工 | 绿色 |
| 加工中 | 蓝色 |
| 待加工 | 灰色 |
| 超时预警 | 后台按组件语义使用橙色或红色，大屏统一亮橙色 |

### 2.3 组件

- 后台表单、表格、分页、弹窗、按钮、菜单和状态标签使用 Ant Design。
- 全局主题统一维护在 `apps/web/src/theme.ts`，优先修改主题 token，不逐页覆盖焦点和链接颜色。
- 卡片圆角不超过 8px，不使用卡片套卡片。
- 后台内容区不使用大面积渐变、发光阴影或装饰图形。
- 表单输入框只显示明确占位文案，例如“请输入产品型号”，不使用“例如……”作为占位内容。
- 普通文本输入框关闭浏览器无关的自动填充或历史建议，不能出现业务未提供的下拉选项。
- 图标按钮优先使用 Ant Design Icons 或现有图标库，并提供可识别的标题或提示。

## 3. 登录页

登录页采用独立布局，不显示后台侧边栏和 FHJD logo。

- 顶部标题：“产品加工流程管理系统”。
- 副标题：“北京飞航吉达航空科技有限公司”。
- 管理员登录卡片在页面中部居中，与标题保持充足间距。
- 卡片内只保留账号、密码和登录按钮，不显示说明性口号或其他登录入口。
- 背景可使用轻量网格、线条或工业几何元素，但不能干扰表单阅读。
- 版权信息固定在页面最底部。
- “管理员登录”使用紧凑标题字号，不与系统主标题争夺视觉层级。

## 4. 管理后台

### 4.1 布局

- 左侧导航使用浅色背景，FHJD 图形标识保持克制尺寸。
- 选中菜单使用浅色背景与边框，不使用左侧高亮竖线和阴影。
- 主体使用浅灰页面底和白色内容区，适合长时间办公。
- 页面标题、筛选区、主操作和表格按稳定顺序排列。

### 4.2 表格与筛选

- 整行可进入详情时使用指针光标和统一悬停背景。
- 产品名称和“详情”操作都可进入产品详情，但整行点击是主要快捷方式。
- 操作文字按钮字号、字重和颜色保持一致，危险操作单独使用危险色。
- 产品列表不显示序号列；按关键词、工序和状态筛选。
- 加载动画只覆盖真实请求时间，缓存可用时不阻塞整个页面。
- 空数据使用简洁空状态，不填充 mock 数据。

### 4.3 产品详情

- 顶部突出产品名称、型号、当前工序和状态。
- “手动调整工序”使用清晰的主按钮样式。
- 基础信息展示产品资料、当前工序进入时间、录入时间和更新时间。
- 流转记录采用表格按时间正序展示全部历史。
- 重复扫码使用橙色标签，管理员调整与扫码进入通过来源字段区分。

## 5. 公开大屏

### 5.1 使用方式

- 地址为 `/screen`，无需登录，只读展示。
- 页面不套用后台布局，不显示导航和后台操作。
- 页面固定铺满 `100vw × 100vh`，桌面电视尺寸下页面本身不出现纵向滚动条。
- 进入页面自动尝试全屏；受浏览器限制时，在首次点击或按键后进入全屏。
- 真正无人值守使用浏览器 kiosk 模式。

### 5.2 当前定稿配色

| 元素 | 颜色 |
| --- | --- |
| 页面背景 | `#020b18` |
| 主文字 | `#f5faff` |
| 次要文字 | `#9fb8cf` |
| 超时 | `#ff9f1c` |
| 产品总数卡 | `#0088d6` |
| 加工中卡 | `#336dff` |
| 已完工卡 | `#18a765` |
| 已超时卡 | `#ff9f1c` |
| 打磨 | `#007c89` |
| 装配 | `#2354be` |
| 喷漆 | `#128052` |
| 包覆 | `#7b3ec8` |

页面背景、工序头部和统计卡均使用纯色，不使用渐变。四道工序主体背景透明，外边框颜色与各自头部一致。

### 5.3 顶部布局

顶部固定为三列：

1. 左侧显示“产品加工流程实时看板”和公司名称，左右页面边距与右侧时间一致。
2. 中间四个统计卡以整个屏幕为基准居中，不因左右内容宽度变化而偏移。
3. 右侧显示当前日期时间和弱化的刷新倒计时。

统计卡宽 `126px`、高 `58px`。数字居中，标签位于右下角；时间使用等宽数字且不设置固定容器宽度，避免数字变化带动中间统计卡跳动。

### 5.4 工序面板

- 主体固定展示打磨、装配、喷漆、包覆四列，不展示完工列。
- 工序名称使用 `25px / 500`，避免过粗。
- 右侧显示“在制”和“超时”，说明文字为白色，数字更大且更亮。
- 四个面板内部使用相同的透明背景，区分色只用于头部和外边框。
- 工序列表内容超出时内部滚动，隐藏视觉滚动条。

### 5.5 产品表格

- 每行只显示产品型号和当前工序已用时，两列均居中。
- 行高固定 `33px`，字号 `14px`。
- 正常文字为白色、字重 `300`。
- 超时文字为亮橙色、字重 `400`。
- 表格横线和中间列线使用低对比灰白色。
- 单元格无左右内边距，型号过长时省略，时间列保持稳定宽度。
- 产品按进入时间升序排列，也就是已用时从长到短。
- 数据不足时用空行补满可视高度，保持完整表格网格，不显示“暂无在制产品”。

### 5.6 数据与刷新

- 真实模式展示接口返回的在制产品。
- 模拟模式每道工序 40 条，共 160 条；数据只存在前端代码中，不写入数据库。
- 管理员在“系统设置 - 大屏看板”切换模拟数据开关。
- 大屏每 5 秒定时刷新并显示弱化倒计时。
- 非重复扫码流转后由 SSE 主动触发刷新；倒计时结束时刷新图标播放一次动画。
- 完工扫码后产品必须即时从工序表格中移除。

## 6. 响应式边界

公开大屏首先针对 16:9 电视和桌面浏览器设计。窄屏时允许四列折为两列或一列并恢复页面滚动，但不得出现文字重叠、卡片溢出或控件遮挡。后台页面需支持常见桌面分辨率和基础移动浏览，不以手机作为主要使用场景。

## 7. 维护要求

- 后台全局样式优先维护在 `theme.ts`，页面结构样式维护在 `App.css` 对应业务区域。
- 大屏当前类名前缀为 `.screen-command-*`，不得重新引入已经废弃的大屏样式体系。
- 修改大屏后至少检查 1920×1080、1366×768 和当前电视实际分辨率。
- 视觉修改必须同时检查真实空数据、少量数据、40 条模拟数据和超时数据四种状态。
