# 界面设计规范

## 1. 当前设计定稿

本阶段界面方向分为两类：

- 后台管理：保持现有干净简洁风格，不做大幅重构。只保持文案、间距、表格、按钮和状态标签一致。
- 登录页与大屏看板：重点优化。登录页要专业、克制、有系统感；大屏看板要更炫酷、更像厂房电视上的实时生产数据墙。

## 2. 后台管理界面

后台用于管理员维护产品、工序、扫码枪和预警，设计原则是清晰、稳定、少干扰。

- 左侧导航继续使用浅色背景。
- 页面主体使用浅灰底和白色内容卡片。
- 不增加复杂背景、动效、装饰图案。
- 不做卡片套卡片。
- 表格、表单、按钮继续使用 Ant Design 组件。
- 主色使用 `#126E78`。
- 告警色只用于预警和错误。

## 3. 登录页设计

登录页采用上下分区结构：

- 顶部：系统标题“产品加工流程管理系统”和公司名称“北京飞航吉达航空科技有限公司”。
- 中部：管理员登录表单居中展示。
- 底部：明确版权信息。

登录页只保留管理员账号密码登录，不加入其他登录入口。

视觉要求：

- 专业内网系统气质。
- 背景保持简洁，可以使用轻量线条、网格或工业感几何元素。
- 表单区域保持白底、清爽、可读。
- 圆角不超过 8px。
- 登录页不放 FHJD logo，不放英文副标题。

## 4. 大屏看板设计

大屏页面 `/screen` 不需要登录，电视或任意系统浏览器输入地址即可查看。

视觉方向：

- 纯色深蓝黑背景，当前使用 `#020b18`。
- 高对比数据面板。
- 大数字指标。
- 流程节点突出打磨、装配、喷漆、包覆。
- 亮橙色用于大屏超时预警，当前使用 `#ff9f1c`。
- 青色、蓝色、绿色、紫色用于工序区分和正常数据高亮。
- 绿色用于完工或正常状态。
- 避免普通后台卡片感，避免说明占位、排行占位和低密度装饰。
- 看板主体不使用 Ant Design 布局组件，以手写 DOM 和 CSS 控制视觉细节。

布局要求：

- 顶部左侧：产品加工流程实时看板和公司名称。
- 顶部中间：产品总数、加工中、已完工、已超时四个小统计卡，统计卡以屏幕整体居中。
- 顶部右侧：当前时间和自动刷新倒计时。
- 主体：按打磨、装配、喷漆、包覆分组展示当前正在进行的产品。
- 每个工序面板展示工序名称、在制数量、超时数量。
- 工序内部使用两列表格展示产品型号和已用时。
- 完工工序不在大屏主体中展示。
- 不展示工序数量排行。
- 不展示说明占位区。
- 已超过 72 小时的产品在工序列表中使用亮橙色预警文字。
- 正常产品型号和时间使用白色文字，字重低于超时文字。

## 5. 数据展示规则

- 指标必须同时包含名称、数字和说明。
- 超时预警优先级最高。
- 大屏文字要适合远距离阅读。
- 表格数据不应过密，行高必须稳定。
- 空数据时保留结构，不显示错乱图表。
- 产品详情中的“工序流转记录”使用表格列表展示，不使用单条当前状态说明替代。
- 流转记录列表字段包括序号、经过工序、进入时间、来源、扫码枪或操作来源、扫码内容或说明。
- 来源标签使用“扫码进入”和“管理员调整”。管理员调整用于扫码枪异常、漏扫或误扫后的进度修正。

## 6. 产品详情页规则

- 顶部显示产品名称、产品型号和当前工序。
- “手动调整工序”按钮使用主按钮样式，保证管理员能快速识别。
- 基础信息区展示产品名称、型号、流水号、数量、当前工序、状态、当前工序进入时间、录入时间、更新时间和备注。
- 工序流转记录区展示该产品已经经过的全部工序历史，按进入时间正序排列。
- 空记录时显示“暂无工序流转记录”。

## 7. 状态色

- 待扫码：灰色。
- 加工中：蓝色。
- 已完工：绿色。
- 临近超时：橙色。
- 已超时：大屏使用亮橙色；后台错误、危险操作可使用红色。

## 8. 本轮实施范围

本轮只实施：

- 登录页视觉优化。
- 大屏看板视觉优化。
- 修复相关页面乱码文案。

本轮不实施：

- 后台管理整体改版。
- 新增复杂图表库。
- 二维码生成或打印界面。
- 外部协同入口。
