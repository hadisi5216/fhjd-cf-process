# 流转记录规则补充

## 1. 定义

产品详情中的“工序流转记录”是该产品已经经过的工序历史列表，不是当前工序说明。

系统中的“当前工序”表示产品现在停留在哪个工序；“流转记录”表示产品从录入后进入过哪些工序，以及每次进入工序的时间和来源。

## 2. 列表字段

产品详情页的工序流转记录采用列表展示，至少包含：

- 序号。
- 经过工序。
- 进入时间。
- 来源：扫码进入或管理员调整。
- 扫码枪或操作来源。
- 扫码内容或调整说明。

## 3. 记录生成规则

- 产品到达某工序后，由该工序固定扫码枪扫码，系统生成一条“扫码进入”记录。
- 后台根据扫码枪绑定关系判断工序，根据扫码内容匹配产品。
- 扫码成功后，产品当前工序更新为扫码枪绑定工序，当前工序进入时间更新为扫码时间。
- 管理员在后台手动调整单个产品当前工序时，系统生成一条“管理员调整”记录。
- 手动调整工序用于处理真实生产流程已经进入某工序，但该工序扫码枪异常、漏扫或误扫的情况。例如产品打磨完成后已经进入装配，但装配扫码枪故障，管理员可以将该产品手动调整到装配工序，并留下管理员调整记录。

## 4. 展示规则

- 记录按进入时间正序展示。
- 同一个产品的所有有效扫码记录都应展示。
- 管理员调整记录与扫码记录展示在同一列表中，通过“来源”字段区分。
- 没有流转记录时，显示“暂无工序流转记录”。
