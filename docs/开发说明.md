# 开发说明

## 1. 当前需求口径

当前阶段只做管理员使用的后台系统。

- 登录只做管理员账号密码登录。
- 管理员 JWT 当前有效期为 8 小时。前端刷新时会校验 token 到期时间，受保护接口返回 `401` 时会统一清除 token、跳转登录页并提示重新登录；登录成功后返回原页面。
- 页面不得把鉴权失败静默显示成空数据，所有后台接口统一复用 `services/api.ts` 中的响应拦截处理。
- 后台只有管理员可以使用。
- 产品维护统一放在产品管理。
- 不再保留任务单和旧独立资产台账两个模块。
- 产品支持手动单条录入，暂不做导入。
- 二维码由其他地方单独打印并贴到产品上，本系统不做二维码生成和打印。
- 本系统当前负责扫码识别、产品流转、看板和预警。
- 大屏看板 `/screen` 不登录即可访问，只读展示。

## 2. 目录结构

当前仓库结构：

```text
apps/
  web/        React 前端，包含管理员后台和公开大屏看板
  api/        NestJS 后端，包含管理员登录、产品、工序、扫码枪、流转、预警和看板接口
packages/
  shared/     前后端共享常量和类型
deploy/       Docker Compose、Nginx 和部署配置
scripts/      数据库备份和恢复脚本
docs/         需求、技术、实施和部署文档
```

建议后续模块结构：

```text
apps/api/src/
  auth/        管理员登录、JWT、登录校验
  admins/      管理员账号和密码维护
  products/    产品管理
  processes/   工序管理
  scanners/    扫码枪管理和扫码上报
  flows/       产品流转记录
  warnings/    超时预警
  dashboard/   后台首页和大屏汇总
  prisma/      Prisma 服务

apps/web/src/
  app/         布局、路由、登录保护
  pages/login/
  pages/dashboard/
  pages/products/
  pages/processes/
  pages/scanners/
  pages/warnings/
  pages/screen/
  services/    API 请求封装
  data/        开发阶段临时 mock，后续逐步移除
```

当前大屏看板实现位置：

- 页面组件：`apps/web/src/pages/screen/ScreenPage.tsx`
- 页面样式：`apps/web/src/App.css` 中 `.screen-command-*` 相关样式
- 看板主体使用手写 DOM 和 CSS，不使用 Ant Design 布局组件。
- 时间和刷新图标使用 Ant Design 图标。
- 当前视觉确认阶段保留大屏预览数据，由后台“系统设置 - 大屏看板 - 大屏预览数据”开关控制。
- 大屏通过 `GET /api/settings/public` 读取公开展示配置，不要求登录。

后续代码调整时，旧模块建议处理为：

- `work-orders`：移除。
- `equipment`：迁移为 `products`。
- 二维码生成、二维码打印页面：当前阶段移除或隐藏。

## 3. 技术栈

- 前端：React + TypeScript + Vite + Ant Design
- 后台管理页面主要使用 Ant Design。
- 大屏看板主体使用手写 React 结构和 CSS，ECharts 暂不用于当前 `/screen` 主体。
- 后端：Node.js + NestJS + Prisma
- 数据库：PostgreSQL
- 部署：Docker + Docker Compose

## 4. 常用命令

安装依赖：

```powershell
npm install
```

生成 Prisma Client：

```powershell
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@localhost:5432/fhjd_cf_process?schema=public"
npm run prisma:generate
```

启动前端：

```powershell
npm run dev:web
```

启动后端：

```powershell
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@localhost:5432/fhjd_cf_process?schema=public"
npm run dev:api
```

构建全部项目：

```powershell
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@localhost:5432/fhjd_cf_process?schema=public"
npm run build
```

Docker 部署：

```powershell
Copy-Item .env.example .env
npm run docker:up
```

首次初始化数据库：

```powershell
docker exec fhjd-cf-process-api npx prisma db push
```

初始化默认数据：

```powershell
docker exec fhjd-cf-process-api npm run prisma:seed
```

`prisma:seed` 会写入默认管理员、默认工序和默认扫码枪。默认扫码枪编码如下：

| 编码 | 名称 | 绑定工序 |
| --- | --- | --- |
| `DM` | 打磨扫码枪 | 打磨 |
| `ZP` | 装配扫码枪 | 装配 |
| `PQ` | 喷漆扫码枪 | 喷漆 |
| `BF` | 包覆扫码枪 | 包覆 |

## 5. 目标接口入口

- `POST /api/auth/login`：管理员登录
- `GET /api/admin/profile`：当前管理员信息
- `PUT /api/admin/password`：修改管理员密码
- `GET /api/products`：产品列表
- `POST /api/products`：新增产品
- `GET /api/products/:id`：产品详情
- `PUT /api/products/:id`：编辑产品
- `DELETE /api/products/:id`：删除产品
- `PUT /api/products/:id/process`：手动调整工序，并生成管理员调整流转记录
- `GET /api/products/:id/flows`：产品流转记录
- `GET /api/processes`：工序列表
- `POST /api/processes`：新增工序
- `PUT /api/processes/:id`：编辑工序
- `DELETE /api/processes/:id`：删除工序
- `GET /api/scanners`：扫码枪列表
- `POST /api/scanners`：新增扫码枪
- `PUT /api/scanners/:id`：编辑扫码枪
- `DELETE /api/scanners/:id`：删除扫码枪
- `POST /api/scanners/scan`：HTTP 扫码上报
- `GET /api/dashboard/summary`：后台首页汇总
- `GET /api/screen/summary`：公开大屏汇总
- `GET /api/settings/public`：公开读取大屏展示配置
- `GET /api/settings`：读取系统设置
- `PUT /api/settings`：保存系统设置
- `GET /api/warnings`：预警列表
- `POST /api/warnings/:id/handle`：处理预警

## 6. 流转记录实现说明

产品详情页的工序流转记录为历史列表，不是当前工序字段的重复展示。

后端 `flow_records` 保存产品已经经过的全部工序记录：

- 扫码枪扫码成功后，写入来源为 `SCAN` 的记录。
- 生产现场扫码枪使用 HTTP POST 上报到 `/api/scanners/scan?name=DM` 这类地址，请求体只发送产品型号文本。
- 后台优先根据 URL 参数中的扫码枪标识匹配扫码枪配置，并据此判断产品进入的工序；来源 IP 只作为兜底。
- HTTP 上报是短连接模式，扫码枪管理只展示最近上报时间，不推算在线或离线状态。
- 管理员在产品详情中手动调整工序后，写入来源为 `MANUAL` 的记录。
- `MANUAL` 记录用于扫码枪异常、漏扫或误扫时补录真实生产进度。
- `scannerId` 对扫码记录必填，对管理员调整记录允许为空。
- `note` 用于保存“管理员手动调整工序”等说明。
- `isDuplicate` 标识重复扫码。首次扫码进入工序时为 `false`，同一产品再次扫描同一工序时为 `true`。
- 重复扫码仍写入 `flow_records`，但不更新产品当前工序、当前工序进入时间和状态。

前端产品详情页展示字段：

- 序号。
- 经过工序。
- 记录时间。
- 来源：扫码进入或管理员调整。
- 扫码枪或操作来源。
- 扫码内容或说明。

当前已有旧接口会在后续开发阶段逐步迁移。
