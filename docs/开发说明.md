# 开发说明

## 1. 环境要求

- Node.js 20
- npm 10 或兼容版本
- PostgreSQL 15
- Docker Desktop 与 Docker Compose（推荐）
- Windows PowerShell 7（本项目当前主要开发环境）

## 2. 仓库结构

```text
apps/
  api/
    prisma/               Prisma Schema 与 seed
    src/
      admins/             管理员资料与密码
      auth/               登录、JWT Guard
      dashboard/          汇总接口与 SSE
      prisma/             Prisma 服务
      processes/          工序管理
      products/           产品与手动流转
      realtime/           大屏更新事件
      scanners/           扫码枪与扫码上报
      settings/           系统设置
      warnings/           超时预警
  web/
    public/               favicon 等公开资源
    src/
      app/                路由保护与后台布局
      pages/              各业务页面
      services/           API 封装与鉴权拦截
      utils/              时间和全屏工具
packages/
  shared/                 共享类型与常量
deploy/
  docker-compose.yml      容器编排
  nginx/default.conf      前端、API 与 SSE 代理
scripts/                  数据库备份和恢复
storage/uploads/          本地开发上传附件（图纸和工艺流程，Git 忽略）
docs/                     项目文档
```

## 3. 安装依赖

在仓库根目录执行：

```powershell
npm install
```

项目使用 npm workspaces。除非只调试某个子项目，日常命令均在仓库根目录运行。

## 4. 开发方式

### 4.1 方式一：完整 Docker 环境

适合验证接近生产的前端、后端、数据库和 Nginx：

```powershell
Copy-Item .env.example .env
npm run docker:up
docker exec fhjd-cf-process-api npx prisma db push
docker exec fhjd-cf-process-api npm run prisma:seed
```

默认访问地址为 `http://127.0.0.1`。若本机 80 端口已占用，修改 `.env`：

```env
WEB_PORT=8080
```

然后访问 `http://127.0.0.1:8080`。

### 4.2 方式二：前后端热更新

此方式需要本机已有可访问的 PostgreSQL。设置本机数据库连接后启动 API：

```powershell
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@127.0.0.1:5432/fhjd_cf_process?schema=public"
npm run prisma:generate
npm run dev:api
```

另开终端启动前端：

```powershell
npm run dev:web
```

Vite 默认使用 `http://127.0.0.1:5173`，实际端口以终端输出为准。开发代理会把 `/api` 转发至本地后端。

首次使用本机数据库时执行：

```powershell
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@127.0.0.1:5432/fhjd_cf_process?schema=public"
npx prisma db push --config apps/api/prisma.config.ts
npm run prisma:seed -w @fhjd-cf/api
```

## 5. 数据库与种子数据

### 5.1 Docker 数据库初始化

```powershell
docker exec fhjd-cf-process-api npx prisma db push
docker exec fhjd-cf-process-api npm run prisma:seed
```

注意脚本名是 `prisma:seed`，不是 `prisma:see`。修改 `schema.prisma` 后必须先执行 `db push`，否则 seed 可能提示数据表不存在。

产品图纸和工艺流程附件分别使用 `product_drawings`、`product_process_attachments` 表。更新代码后必须执行 `db push`，并确认 `UPLOAD_DIR` 指向后端进程可写目录。本地不设置时默认使用 `apps/api/storage/uploads`。

### 5.2 默认数据

seed 会以 upsert 方式写入：

- 管理员：`admin`，新建时初始密码 `admin123456`。
- 工序：打磨、装配、喷漆、包覆、完工。
- 扫码枪：`DM`、`ZP`、`PQ`、`BF`、`WG`，默认 IP 为 `192.168.1.11` 至 `192.168.1.15`。
- 系统设置：默认开启大屏模拟数据。

再次执行 seed 不会覆盖已有管理员密码。首次部署后应立即登录修改初始密码，并在设置中关闭大屏模拟数据。

### 5.3 Schema 变更流程

当前项目尚未固化正式 migration，开发和当前部署阶段使用：

```powershell
docker exec fhjd-cf-process-api npx prisma db push
```

进入正式版本管理后应生成 migration，并在服务器使用 `prisma migrate deploy`。不要在生产库直接使用 `prisma migrate dev`。

## 7. 扫码接口调试

### 7.1 PowerShell

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1/api/scanners/scan?name=DM" `
  -ContentType "text/plain" `
  -Body "J/CLL9-12A-101S1.1"
```

本地开发前端使用 5173、API 直接运行在 3000 时，可将地址改为 `http://127.0.0.1:3000/api/scanners/scan?name=DM`。

### 7.2 curl

```bash
curl -X POST \
  -H "Content-Type: text/plain" \
  --data "J/CLL9-12A-101S1.1" \
  "http://127.0.0.1/api/scanners/scan?name=DM"
```

成功响应中的 `duplicated` 表示是否为当前工序重复扫码，`message` 为“流转成功”或“重复扫码已记录”。

### 7.3 后端日志

```powershell
docker logs -f fhjd-cf-process-api
```

扫码无流转时依次检查：请求是否到达、URL 编号是否匹配启用扫码枪、产品型号是否完全一致、数据库表是否已同步。

## 8. 接口与鉴权约定

- API 全局前缀为 `/api`。
- 管理接口统一使用 JWT Bearer Token。
- `/api/scanners/scan`、`/api/dashboard/summary`、`/api/dashboard/events`、`/api/settings/public` 和 `/api/health` 为公开接口。
- 新增后台请求必须复用 `apps/web/src/services/api.ts`，避免绕过统一鉴权失效处理。
- 日期字段由后端使用 ISO 8601 UTC 返回，前端通过 `utils/datetime.ts` 转换为本地时间显示。

## 9. 开发约束

- 业务名称统一使用“产品”，不再使用“设备”或“任务单”。
- 工序排序只用于显示，不得用于限制扫码顺序。
- 修改扫码逻辑时必须覆盖同工序重复、跨工序流转、返回旧工序、完工和完工后返工场景。
- 流转记录只能追加，不得因修改当前工序而删除历史。
- 大屏保持 5 秒轮询，扫码主动刷新是附加能力，不能替代轮询兜底。
- 大屏模拟数据只能由设置开关启用，不能写入数据库或混入真实产品接口。
- 后台使用 Ant Design；公开大屏主体保持手写结构和样式。
- 工艺流程附件仅用于制作参考，不得接入扫码、流转、超时或看板统计逻辑。
- 产品导出必须复用产品列表筛选条件，但不得应用列表的 200 条展示上限；Excel 包含产品档案、工艺附件名称、当前工序、状态和时间字段。
- 图纸和工艺流程附件本体不写入数据库和 Git；接口只保存随机存储名，页面展示原始文件名。
