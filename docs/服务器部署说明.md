# 服务器部署说明

本文档适用于新服务器已经安装 Docker 和 Docker Compose 的情况。

## 1. 部署目录

建议把项目放在固定目录：

```bash
mkdir -p /opt/fhjd-cf-process
cd /opt/fhjd-cf-process
```

上传代码后，目录应包含：

```text
apps/
packages/
deploy/
docs/
package.json
package-lock.json
.env.example
```

## 2. 环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env`：

```bash
vi .env
```

生产环境至少修改：

```env
POSTGRES_DB=fhjd_cf_process
POSTGRES_USER=fhjd_cf
POSTGRES_PASSWORD=请改成强密码
DATABASE_URL=postgresql://fhjd_cf:请改成强密码@postgres:5432/fhjd_cf_process?schema=public
JWT_SECRET=请改成较长随机字符串
WEB_PORT=80
```

`POSTGRES_PASSWORD` 与 `DATABASE_URL` 中的密码必须一致。

如果服务器拉取 Docker Hub 镜像不稳定，可在 `.env` 中指定镜像和 npm 源：

```env
POSTGRES_IMAGE=postgres:15-alpine
REDIS_IMAGE=redis:7-alpine
NODE_IMAGE=node:20-alpine
NGINX_IMAGE=nginx:1.27-alpine
NPM_REGISTRY=https://registry.npmmirror.com/
```

## 3. 启动系统

在项目根目录执行：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml up -d --build
```

查看容器：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml ps
```

容器名称应为：

```text
fhjd-cf-process-web
fhjd-cf-process-api
fhjd-cf-process-postgres
fhjd-cf-process-redis
```

其中 `fhjd-cf-process-web` 是前端 Nginx 容器，也是浏览器访问入口。

## 4. 初始化数据库

当前阶段使用 Prisma `db push` 创建数据库表：

```bash
docker exec fhjd-cf-process-api npx prisma db push
```

流转记录规则已包含以下数据库结构：`flow_records.source`、`flow_records.note`、重复扫码标记 `flow_records.is_duplicate`，以及允许管理员调整记录的 `flow_records.scanner_id` 为空。新服务器部署或已有库升级后都需要执行一次 `prisma db push`，确保表结构同步。

当前系统设置已包含 `system_settings` 表，用于保存大屏预览数据开关等配置。新服务器部署或升级后同样需要执行 `prisma db push`。

初始化默认数据：

```bash
docker exec fhjd-cf-process-api npm run prisma:seed
```

默认数据包含：

- 管理员账号 `admin`
- 默认工序：打磨、装配、喷漆、包覆、完工
- 默认扫码枪：`DM`、`ZP`、`PQ`、`BF`
- 默认系统设置：大屏预览数据开启

默认管理员账号：

```text
账号：admin
密码：admin123456
```

上线后请立即在后台修改密码，并在“扫码枪管理”中补充现场扫码枪的固定 IP 或网络标识。

## 5. 访问地址

假设服务器 IP 为 `192.168.1.100`，`.env` 中 `WEB_PORT=80`：

```text
后台管理：http://192.168.1.100/dashboard
大屏看板：http://192.168.1.100/screen
```

后台管理需要管理员登录。大屏看板 `/screen` 不需要登录，电视或任意系统的浏览器输入地址即可展示。

从后台点击“大屏看板”或“打开大屏”时，系统会直接请求浏览器进入全屏。直接在地址栏打开 `/screen` 时会自动尝试全屏；如果浏览器安全策略要求用户操作，则首次点击或按键后进入全屏。

电视开机后需要完全无人值守地自动全屏时，应将电视浏览器或主机浏览器配置为 kiosk 模式，并将 `/screen` 设置为启动页。网页自身无法绕过浏览器的全屏安全限制。

## 6. 常用运维命令

查看运行状态：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml ps
```

查看日志：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml logs -f
```

只看后端日志：

```bash
docker logs -f fhjd-cf-process-api
```

重启系统：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml restart
```

停止系统：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml down
```

停止并删除数据库数据卷需谨慎，生产环境一般不要执行：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml down -v
```

## 7. 更新版本

上传新代码后执行：

```bash
docker compose -p fhjd-cf-process -f deploy/docker-compose.yml up -d --build
docker exec fhjd-cf-process-api npx prisma db push
```

如果后续项目已有正式 migrations，第二行改为：

```bash
docker exec fhjd-cf-process-api npx prisma migrate deploy
```

## 8. 端口说明

- `WEB_PORT`：前端访问端口，默认 `80`。使用默认值时，浏览器直接输入服务器 IP 即可访问，不需要额外输入端口。
- 扫码枪使用 HTTP POST 上报，默认走 `WEB_PORT` 对应的 HTTP 端口。
- 后端接口监听容器内 `3000`，不映射到宿主机端口。前端 Nginx 通过 Docker 内网代理 `/api/` 到后端，因此不会占用服务器的 `3000` 端口。
- PostgreSQL 和 Redis 只在 Docker 内网中使用，不映射到宿主机端口。

## 9. 防火墙

如果服务器开启防火墙，需要放行 HTTP 默认端口 `80`：

```bash
firewall-cmd --add-service=http --permanent
firewall-cmd --reload
```

Ubuntu 常见命令：

```bash
ufw allow 80/tcp
```

如果服务器上已经有其他 Nginx、Apache、IIS 或旧容器占用 `80` 端口，需要先停止占用端口的服务，或临时把 `.env` 中 `WEB_PORT` 改为其他端口。

扫码枪配置要求：

- 协议：HTTP
- 请求方式：POST
- HTTP 版本：1.1
- 服务器地址：部署服务器 IP
- 服务器端口：`80`
- 请求路径：`/api/scanners/scan?name=DM`
- 请求头：`Content-Type: text/plain`
- 上报内容：只发送产品型号文本作为请求体
- 后台“扫码枪管理”中建议把每台扫码枪编号维护成简短稳定标识，例如 `DM`、`ZP`、`PQ`、`BF`。URL 参数 `name=DM` 会优先匹配扫码枪编号或扫码枪名称，系统据此判断该扫码枪绑定的工序。
- 如果 URL 未携带扫码枪标识，系统才尝试使用来源 IP 匹配后台扫码枪 IP。
- HTTP 为短连接上报，后台只记录和展示最近上报时间，不推算在线或离线状态。

## 10. 常见问题

### 10.1 拉取镜像时报 EOF

先单独测试镜像能否拉取：

```bash
docker pull postgres:15-alpine
docker pull redis:7-alpine
docker pull node:20-alpine
docker pull nginx:1.27-alpine
```

如果服务器有内网镜像仓库，可在 `.env` 中修改镜像地址后重新执行启动命令。

### 10.2 npm install / npm ci 网络失败

在 `.env` 中设置更稳定的 npm 源：

```env
NPM_REGISTRY=https://registry.npmmirror.com/
```

### 10.3 后台接口 500，提示数据库表不存在

说明数据库尚未初始化或模型变更后没有同步，执行：

```bash
docker exec fhjd-cf-process-api npx prisma db push
docker exec fhjd-cf-process-api npm run prisma:seed
```

然后检查：

```bash
curl http://服务器IP/api/health
curl http://服务器IP/api/dashboard/summary
```
