# 产品加工流程管理系统

本项目用于建设一套面向北京飞航吉达航空科技有限公司厂房内部使用的产品加工流程管理系统。系统通过“产品二维码识别 + 固定工位扫码枪 + 后台管理 + 实时看板 + 超时预警”，让产品在打磨、装配、喷漆、包覆等工序中的当前位置、进入时间、滞留时长和预警状态清晰可见。

## 当前需求口径

- 系统后台只给管理员使用。
- 登录只做管理员账号密码登录和密码维护。
- 不做工序负责人登录，不做多角色权限体系。
- 原任务单和旧独立资产模块统一调整为“产品管理”。
- 产品支持管理员手动单条录入，暂不做批量导入。
- 二维码由其他地方单独打印并贴到产品上，本系统不做二维码生成、二维码打印或标签模板。
- 本系统当前负责扫码识别、产品流转记录、实时看板和超时预警。
- 产品详情中的工序流转记录为历史列表，展示产品已经经过的全部工序。
- 当产品真实进入某工序但扫码枪异常、漏扫或误扫时，管理员可在后台手动调整工序，并留下“管理员调整”记录。
- 大屏看板 `/screen` 不需要登录，任意浏览器输入地址即可只读展示。

## 默认工序

1. 打磨
2. 装配
3. 喷漆
4. 包覆
5. 完工

## 产品字段

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| 序号 | 内部排序或清单序号 | 1 |
| 产品名称 | 产品或部件名称 | 12号舵舱体 |
| 产品型号 | 产品或部件型号，当前作为扫码识别主要内容 | J/CL19-12A-101S1.1 |
| 流水号 | 批次或生产流水号 | 2606003 |
| 数量 | 数量 | 1 |
| 单位 | 计量单位 | 件 |
| 备注 | 补充说明 | 空 |

## 核心流程

1. 管理员登录后台。
2. 管理员在产品管理中手动录入产品。
3. 产品二维码由外部方式提前打印并贴好。
4. 产品到达某个工序后，现场人员使用该工序固定扫码枪扫码。
5. 后台根据扫码枪编号或 IP 判断产品进入的工序。
6. 后台根据扫码内容匹配产品。
7. 系统记录产品进入该工序的时间并更新当前状态。
8. 产品详情页展示该产品已经经过的全部工序流转记录。
9. 如果扫码枪故障导致真实进度无法扫码记录，管理员可手动调整工序，系统记录为“管理员调整”。
10. 产品在当前工序滞留超过 72 小时，系统自动预警。

## 当前看板口径

- `/screen` 为公开只读大屏页面，不需要登录。
- 看板当前采用纯色深蓝黑背景，四个顶部统计卡居中显示。
- 工序区域展示打磨、装配、喷漆、包覆四个工序。
- 每个工序顶部展示工序名称、在制数量、超时数量。
- 工序内部使用两列表格展示产品型号和当前工序已用时。
- 正常产品使用白色文字，超时产品使用亮橙色文字。
- 工序列表隐藏滚动条，但保留内部滚动能力。
- 当前视觉确认阶段保留大屏预览数据，正式联调真实接口时关闭预览开关。
