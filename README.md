# 产品加工流程管理系统

北京飞航吉达航空科技有限公司厂房内部使用的产品加工流程管理系统。系统通过固定工位扫码枪接收产品型号，记录产品在打磨、装配、喷漆、包覆和完工工序中的流转情况，并提供后台管理、实时看板与超时预警。

## 当前范围

- 后台仅供系统管理员使用，支持账号密码登录与密码修改。
- 产品由管理员手动单条录入，当前不提供批量导入。
- 产品列表支持按当前关键词、工序和状态筛选结果批量导出 Excel。
- 产品详情支持上传 DOCX、XLSX 工艺流程附件，可在线查看、下载和删除，不参与扫码流转。
- 产品详情支持上传多份图纸；PDF 可在新标签在线预览，其他格式可下载。
- 二维码由外部系统生成并打印，本系统只负责识别扫码枪上报的产品型号。
- 各工序没有固定先后顺序，扫描哪道工序的扫码枪即进入哪道工序。
- 同一产品连续扫描当前工序时记为重复扫码，保留记录但不重置进入时间。
- 产品切换工序后再返回原工序，视为重新进入并重新计时。
- 完工扫码枪将产品置为已完工，并立即从大屏在制列表移除。
- 管理员可在产品详情中手动调整工序，系统保留“管理员调整”记录。
- 工序默认时限为 72 小时，超时后进入预警中心并在看板中高亮。
- 公开大屏 `/screen` 无需登录，支持扫码事件即时刷新和 5 秒定时兜底刷新。

## 技术栈

| 层级 | 技术 |
| --- | --- |
| 前端 | React 19、TypeScript、Vite、Ant Design、TanStack Query |
| 后端 | Node.js 20、NestJS 11、Prisma 7 |
| 数据库 | PostgreSQL 15 |
| 实时更新 | Server-Sent Events（SSE） |
| 部署 | Docker、Docker Compose、Nginx |

## 项目结构

```text
fhjd-cf-process/
  apps/
    api/              NestJS 接口服务
    web/              React 管理后台和公开大屏
  packages/shared/    前后端共享类型和常量
  deploy/             Docker Compose 与 Nginx 配置
  scripts/            数据库备份、恢复脚本
  docs/               项目文档
```

## 快速开始

安装依赖并启动本地前后端：

```powershell
npm install
$env:DATABASE_URL="postgresql://fhjd_cf:change_me@localhost:5432/fhjd_cf_process?schema=public"
npm run dev:api
```

另开一个终端：

```powershell
npm run dev:web
```

完整的本地数据库准备、种子数据和测试命令见[开发说明](./docs/开发说明.md)。服务器部署见[服务器部署说明](./docs/服务器部署说明.md)。
